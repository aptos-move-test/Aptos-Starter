### Prepare aptos-core[](https://aptos.dev/cli-tools/aptos-cli-tool/use-aptos-cli#prepare-aptos-core)

### TheÂ `layout`Â file[](https://aptos.dev/cli-tools/aptos-cli-tool/use-aptos-cli#the-layout-file)

The layout file contains:

- `root_key`: an Ed25519 public key for AptosCoin management.
- `users`: the set of participants
- `chain_id`: theÂ `ChainId`Â or a unique integer that distinguishes this deployment from other Aptos networks

An example:


### TheÂ `ValidatorConfiguration`Â file[](https://aptos.dev/cli-tools/aptos-cli-tool/use-aptos-cli#the-validatorconfiguration-file)

- `account_address`: The account that manages this validator. This must be derived from theÂ `account_key`Â provided within theÂ `ValidatorConfiguration`Â file.
- `consensus_key`: The public key for authenticating consensus messages from the validator

- `network_key`: The public key for both validator and fullnode network authentication and encryption.
- `validator_host`: The network address where the validator resides. This contains aÂ `host`Â andÂ `port`Â field. TheÂ `host`Â should either be a DNS name or an IP address. Currently only IPv4 is supported.

1. Generate yourÂ `ValidatorConfiguration`:

### Generating a genesis and waypoint[](https://aptos.dev/cli-tools/aptos-cli-tool/use-aptos-cli#generating-a-genesis-and-waypoint)
